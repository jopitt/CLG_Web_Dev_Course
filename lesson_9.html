Form Submission

  • At the end of a form, we can display a button for a user to click on once they've finished filling out the form. Click the button submits the user's responses to a server or endpoint for processing.
  • To display a submit button we can use an input element with a type attribute of submit, e.g.
  • There are several attributes on the form element which control the way a form is submitted. The 2 most important attributes are the action and method attributes.
  • The action takes care of where form data is sent, while the method defines how, e.g.
  • The action attribute usually points to a webpage or URL that has been designed to process the form data we're sending.
  • The method attribue controls what type of request is used to send the data. The value of the method can be either get or post. This is because get and post are the 2 types of HTTP web requests we can send.
  • HTTP is a communications protocol - i.e. an agreed upon standard way of communicating - between the client (user device) and server (the machine or machines wehre backend software is hosted.
  • To send data via HTTP we send a request. When we submit an HTML form, we tell our webpage to send the data as a HTTP request.
  • A post request is used when we want to post data to server. We can send quite a lot of data in a post request.
  • A get request is usually what we use when we want to retrieve data from the server. Get requests are smaller than post requests.
  • If we want to retrieve information, like when submitting a search form, we usually use a get request.
  • If we want to send a lot of information, such as filling out a job application, we usually use a postrequest.

Lesson 7: CSS conflicts

Inheritance

  • As we get more advanced with our CSS, how the browser decides which CSS rule to apply becomes more complex.
  • If no value is set for a property on an element, that property inherits the property value of its parent element.
    This is unless it's a non-inherited property, which will take on the initial value of that property.
  • For example, if we have a parent and child element with a colour and border set on the parent:
    • the interited color property vlaue would apply to both parent and child, but
    • the non-inherited border would only apply to the parent.

Positioning and specificity

  • The position of your CSS rules matter if you have competing CSS styles.
  • The latest rule declared in a stylesheet will override any of those that come before it.
  • CSS has various ways you can apply specificity to styling.
    The more specific a selector is, the higher importance it gets graned by your browser.
  • Later CSS selectors may be overridden by earlier ones if they are more specific.
  • Earlier selectors will take precedence only if they're more specific.
  • Order of specificity of selectors:
    • IDs
    • classes
    • pure element

    e.g.
    #id { color: red; }
    .class { color: blue; }
    p { color: yellow; }

Importance

  • The !important value overrides all other rules though we should only use it when we have no other option.
  • Using !important is considered bad development as stylesheets get messy, unreadable and unmaintainable.

Lesson 4: Divs, spans, IDs and classes

Formatting tags

  • Some HTML elements are used primarily for formatting:
    • sub for subscript testlike this
    • sup for superscript textlike this
    • del for deleted text
    • ins for inserted text
    • b for bold text
    • i for italised text
    • u for underlined text.
  • Modern web pages have mainly moved away from using b, i and u tags and instead use:
    • em for emphasis, styled italic by default
    • strong for important, styled bold by default
  • The b and i tags can still be used, but they are seen less often and used only when other tags aren't appropriate.

Divs, spans, IDs and Classes

  • Div and span are generic elements used to help style HTML.
  • Div elements are block level elements. They allow you to group html elements together.
  • Span elements are inline elements. They allow you to apply different styles to parts of elements.
  • A block level element:
    • starts on a new line
    • takes up the full width of its parent element
    • has the elements after it start on the next line.
  • You can set the width and height of block level elements.
  • An inline element:
    • does not have its own line
    • is the width of its contents.
  • You can't set the width and height of inline elements.
  • An ID uniquely identifies an element on a page. We style an element with a particular ID using the CSS selector # followed by the ID, e.g.
    #introParagraph {
    color: blue;
    }
  • A class attribute identifies elements that have something in common with other elements with the same class.
  • We style elements of the same class using the CSS selector . followed by the class name, e.g.
    .coupon {
    border: dashed;
    }

Background styling

  • We can control the background of an element using the following properties:
    • background-color
    • background-image url('imagepath')
    • background-repeat repeat-x, repeat-y, repeat or no-repeat
    • background-position left or right and top or bottom
    • background-size:
      • contain (scales as large as possible within its container without cropping or stretching)
      • cover (scales the image as large as possible to fill the container, stretching if necessary)
      • auto (scales the background image in the corresponding direction such that its intrinsic proportions are maintained)
      • "length" (stretches the image in the corresponding dimension to the specified length)
      • "percentage" (stretches the image in the corresponding dimension to the specified percentage of the background positioning area)
  • Or, to keep our CSS neater, we can use the background shorthand property, e.g.
    .container {
    background: #ff6347 url("school.png") no-repeat top center;
    }

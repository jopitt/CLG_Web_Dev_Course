Lesson 6: Modern HTML & CSS

HTML Semantics

  • HTML5 is the latest version of HTML.
  • HTML5 adds new capabilities to the HTML such as new elements used for multimedia.
  • It also includes new semantic elements such as footer, nav and article, which better describe the meaning of the content on the page.
  • We use the header tag and footer tag to represent our site's main header and footer respectively (used only once on the webpage).
  • The nav element can wrap a site's navigation. There could be multiple sets of navigational content:
    • in the header
    • another one in the sidebar
    • a smaller one in the footer.
  • There are also several semantic elements to wrap certain types of page content too, e.g.
    Element Function
    article usually wraps smaller independent pieces of content (articles are self-contained material and shouldn't need an context)
    section represents a new segment on your page and acts like a divider

CSS Flexbox

  • Flexbox is a set of CSS rules that provides a more efficient way to lay out, align and distribute space and items in a container.
    It allows for more complete web page designs by challenging the default block and inline layouts.
    Flexbox works horizontally and/or vertically.
  • To start a flexbox layout, all you need is a parent element (container) with at least 1 child.
    The display property enables a flexbox layout.
    Changing the display property on that container enables flexible content for all its direct children, e.g.
    .container {
    display: flex;
    }
    By default, if you only set display: flex, all flex items will be positioned on 1 line within their parent container.
  • The flex-direction sets the diretion items are placed in the container, e.g.
    .container {
    flex-direction: row | row-reverse | column | column-reverse;
    }
    Property Item layout
    row default property - items are placed the same as the text direction
    row-reverse items are placed opposite to the text direction
    column items are placed top to bottom
    column-reverse itmes are plaed bottom to top
  • The other axis to consider when working with flexbox is the cross axis. This is perpendicular to what you define in flex-direction.
    By default, this would be vertical.
  • The flex-wrap property will adjust how the items wrap within the container, e.g.
    .container {
    flex-wrap: nowrap | wrap | wrap-reverse;
    }
    Property Item layout
    no wrap default property - every item is fit to a single line
    wrap items wrap to additional lines
    wrap-reverse items wrap around to additional lines in reverse
  • The flex-flow property combines the properties flex-direction and flex-wrap, e.g.
    .container {
    flex-flow: column wrap;
    }
  • The justify-content property aligns items horizontally (or vertically when flex direction in column), e.g.
    .container {
    justify-content: flex-start | flex-end | center | space-between | space-around | space-evenly;
    }
    Property Item layout
    flex-start default property - items align to the left side of the container
    flex-end items align to the right side of the container
    center items align at the center of the container
    space-between items display with equal spacing between them
    space-around items display with equal spacing around them
    space-evenly does a smart little space calculation and evenly distributes left-over space between the items and the edges
  • The align-items property aligns items vertically (or horizontally if flex direction is column) and align-self can be used for a specific item, e.g.
    .container {
    align-items: stretch | flex-start | flex-end | center;
    }
    or
    .yellow {
    align-self: stretch | flex-start | flex-end | center;
    }
    Property Item layout
    stretch default property - items are stretched to fit the container
    flex-start items align to the top of the container
    flex-end items align to the bottom of the container
    center items align at the vertical centre of the container
    baseline items display at the baseline of the container
  • The align-content property sets how multiple lines are spaced apart from each other, e.g.
    .container {
    align-content: stretch | flex-start | flex-end | center | space-between | space-around | space-evenly;
    }
    Property Item layout
    stretch default property - lines display with equal spacing around them
    flex-start lines are packed at the top of the container
    flex-end lies are packed at the vertical center of the container
    center lines are packed at the vertical center of the container
    space-between lines display with equal spacing between them
    space-around lines display with equal spacing around them
    space-evenly evenly distributes the items in the remaining space of the container
  • align-content determines the spacing between lines, while align-items determines how the items as a whole are aligned within the container. When there is only 1 line, align-content has no effect.
  • order properties can be assigned to individual items, e.g.
    .yellow {
    order: (... -1, 0 (default), 1, ...);
    }
